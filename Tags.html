Semantiken

Strukturelemente zur Unterteilung der Website

Zeigt an welches der Hauptcontent der Website ist.
Der Header befindet sich ganz oben auf der Website. Beinhaltet meist Navigationselemente wie Links oder das Logo.
Der Footer befindet sich ganz unten auf der Website. Beinhaltet meist Kontaktdaten, Links, Copyright, Impressum, etc.
Nav dient zum erstellen einer Navigationsleiste.
Eine Section wird genutzt um header und footer einzurahmen.
In einem Article wird unabhängiger Content deklariert.
Ein Div ist ein Behältnis für unabhängigen Content und zur Einteilung.

Listen

Es gibt unterschiedliche Arten von Listen: ungeordnete, geordnete und beschreibende.

Die ungeordnete Liste

  • Das erste Element der Liste.
  • Bei einer ungeordneten Liste werden Aufzählungszeichen statt Zahlen verwendet.
  • Diese Aufzählungszeichen kann man natürlich auch später noch in CSS ändern.

Die geordnete Liste

  1. Hier wird mit Zahlen statt Zeichen aufgezählt.
  2. Die kann man auch ändern, wenn man etwas anderes dort stehen haben will.
  3. Aber so ist es am Anfang am leichtesten zu verstehen.

Die beschreibende Liste

Was ist das?
- Eine Liste mit Description.
Was macht man da rein?
- Dinge die man beschreiben möchte.

Medien

Um Medien auf einer Website einzufügen gibt es folgende Möglichkeiten:

Bilder

Das "alt"-Attribut dient hierbei vor allem der Usability (HCI im 2. Semester). Dadurch können z.b. Blinde mit ihren Screenreadern erfahren was auf einem Bild zu sehen ist.
Das "src"-Attribut spezifiziert die Herkunft des Bildes. Entweder aus einem Lokalen Ordner, oder von einer Fremdwebsite. Funktioniert mit Formaten wie gif, svg, png, jpg etc.

Videos

Videos sind recht ähnlich zu Bildern. Man braucht eine src, aber dazu auch noch den type in dem das Format angegeben wird. Das obere Video wird als autoplay und gemuted abgespielt,
Wie die ganzen nervigen Werbungen immer. Außerdem ist die Größe des Fensters angegeben (kann man aber auch mit CSS machen).

Audio

Audios sind recht ähnlich zu Videos. Man braucht eine src, aber dazu auch noch den type in dem das Format angegeben wird. Controls gibt dann sowas wie play, pause und volume an.

Weitere Attribute für Audio und Video.

autoplay, buffered, controls, crossorigin, height, loop, muted, played, preload,poster, src, width. Wie gut diese allerdings funktionieren kommt auf den Browser an.
